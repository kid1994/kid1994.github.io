对其它对象提供一种代理以便控制对该对象的访问。

原理分析

概述

首先，程序在运行时会动态地生成一个代理类，这个代理类要实现一组接口，称为代理接口(proxy interfaces)。代理对象就是这个代理类的一个实例。

每个代理对象都有一个关联的invocation handler，这个handler实现了InvocationHandler接口。当触发代理对象的方法时，其实是派发给这个handler的invoke方法去执行的。看一下这个invoke方法的签名：

代理类拥有如下特性：

  • 代理类的名字一般以$Proxy开头，比如上节示例中的proxyHello的类名字为class edu.xidian.howtodoinjava.reflection.$Proxy0
  • 代理类是final和non-abstract的
  • 每个代理类都继承了java.lang.reflect.Proxy类
  • 代理类会实现所有代理接口，并且是按照创建时所指定的顺序
  • Proxy.isProxyClass方法可以用来判断一个类是否是代理类

代理对象拥有如下特性：

  • 如果proxy是代理对象，Foo是代理接口，那么proxy instanceof Foo会返回true；并且(Foo) proxy可以成功转化
  • 每个代理对象都有一个关联的invocation handler

代理类所属的包和模块，以及可访问性，由它所实现的代理接口所决定。
