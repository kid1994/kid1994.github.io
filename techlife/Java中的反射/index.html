Java中的反射机制为程序提供了一种运行时自省（introspect)的能力。利用反射，我们可以在运行时获取类的信息，直接操作类和对象，甚至动态地修改类的定义。

反射的应用场景有：

  • IDE中的浏览类信息，代码审查等功能
  • 调试程序，会需要获取类的私有成员信息等
  • 单元测试框架，比如触发以test开头的方法等
  • 动态代理，JDK Proxy
  • 依赖注入框架
  • ORM框架

当然，在带来强大功能的同时，反射机制也存在着一些问题：

  • 性能损耗，需要进行访问权限验证；一些编译器的优化在反射中无法执行
  • 安全问题，反射操作需要运行时权限
  • 暴露了内部状态，破坏了封装性，会带来一系列的副作用

反射操作概览

获取类对象

所有反射操作的入口都是java.lang.Class类对象，所以我们首先要获取类对象。

获取类对象有以下几种方式：

  • 如果已经有了对象实例，则可以通过Object.getClass()方法来获取类对象。只适用于引用类型。
  • 如果已经有了数据类型，则可以通过.class语法来获取类对象。适用于引用类型和原子类型。
  • 如果已经有了类的全限定名，则可以通过Class.forName()静态方法来获取类对象。只适用于引用类型。
  • 对于原子类型，除了.class语法，还可以通过对应的引用类型的.TYPE语法来获取类对象。
  • 如果已经有了类对象，则可以通过一些返回类对象的反射API来获取其它的类对象。

获取类的成员信息

有了类对象后，我们还可以获取类的成员信息，包括：

  • 类的域
  • 类的方法
  • 类的构造方法

获取类的域的方法有：

  • getDeclaredField()
  • getField()
  • getDeclaredFields()
  • getFields()

获取类的方法的方法有：

  • getDeclaredMethod()
  • getMethod()
  • getDeclaredMethods()
  • getMethods()

获取类的构造方法的方法有：

  • getDeclaredConstructor()
  • getConstructor()
  • getDeclaredConstructors()
  • getConstructors()

关于这些方法之间的区别：

  • 有无Declared：有则表示只获取私有成员信息；无则表示获取本类的公有成员以及从父类继承过来的成员信息。
  • 单复数：表示是否获取所有的成员信息。

构造方法

有两种通过反射创建一个新的实例对象的方法：

  • java.lang.reflect.Constructor.newInstance()
  • Class.newInstance()

推荐使用前一种方法，因为：

  • 后者只能调用无参数的构造方法；而前者可以调用任意一个构造方法
  • 后者会将构造方法的异常抛出；而前者会将异常封装在InvocationTargetException中
  • 后者必须要求构造方法是公有的；而前者可以调用私有的构造方法
