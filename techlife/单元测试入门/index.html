单元测试是指对系统中的功能单元进行测试。维基百科上的解释是：单元测试（又称为模块测试, Unit Testing）是针对程序模块（软件设计的最小单位）来进行正确性检验的测试工作。程序单元是应用的最小可测试部件。在过程化编程中，一个单元就是单个程序、函数、过程等；对于面向对象编程，最小单元就是方法，包括基类（超类）、抽象类、或者派生类（子类）中的方法。

单测的必要性

个人认为单测的必要性有以下几点：

  1. 保证代码的质量。这点不言自明，单测可以保证代码正确地实现了功能。
  2. 提高开发的效率。虽然表面上编写单测代码可能会花更多的时间，但从长远的角度来看，实际上是能够提高开发效率的。因为保证了一个功能单元的正确和可靠性，将不会威胁到其它功能单元的运行，在出现了bug时也能更方便地调试。
  3. 降低项目的风险。单测能够尽早地发现bug，避免了bug的堆积为项目带来的风险。

单测的难点

单测虽然带来了诸多好处，但实施的时候也会面临诸多难点：

  1. 学习成本。在单测的过程中会面临许多的问题场景，为了解决这些问题，我们需要额外学习一些工具和框架。比如为了避免外部依赖的影响，我们需要学习mock框架；为了避免产生脏数据，我们需要学习内存数据库等。
  2. 开发成本。为功能单元编写全面的测试用例，无疑会增加团队的开发成本。

注解

Junit提供了一系列注解帮助我们执行测试，除了上述@Test注解外，还有以下几种:

  1. @Ignore : 可用于标注类或方法，表示忽略该类或方法的测试。
  2. @BeforeClass : 用于标注方法，表示该方法需要在所有单测方法执行前执行一次，通常用于执行一些初始化的工作。该注解标注的方法必须是public static void且无参数的,通常命名为setup。
  3. @AfterClass : 用于标注方法，表示该方法需要在所有单测方法执行完后再执行，通常用于执行一些回收资源之类的工作。该注解标注的方法同样必须是public static void且无参数，通常命名为teardown。
  4. @Before : 用于标注方法，表示该方法在每个单测方法执行前都要执行一次，用于执行一些准备工作。标注的方法必须是public void且无参数的。
  5. @After ： 和@Before对应。

断言

断言是一种自动化测试的机制，表示程序执行的结果要符合某种预期，不然就是失败的。除了上面的assertEquals断言，Junit还提供了针对原子类型、数组和引用类型的各种断言：

  1. assertEquals表示结果要等于某个给定的值。
  2. assertArrayEquals表示一个数组里的值都要等于某个给定数组里对应的值。
  3. assertTrue和assertFalse表示结果要为真/假。
  4. assertNull和assertNotNull表示结果为空或不为空。
  5. assertSame和assertNotSame表示两个对象是否为同一个对象。
  6. assertThat是最复杂也最灵活的一种断言，是在其它断言都不能满足使用的时候用的。它表示断言执行的结果要满足某种条件。该方法的签名如下:

spring test

Spring test是Spring框架中的一个模块，也就是说，它可以使用Spring提供的IOC特性。使用Spring test的原因主要就是项目使用了Spring，所有bean都是由Spring负责注入的，这样我们如果要测试某个类的时候，就不能直接new了，还得从Spring的容器中去获取。

Spring test相当于在Junit之上加入了IoC特性，这样在单测中需要使用的Bean就可以由Spring 帮忙注入了。另一方面，Spring test可以在运行多个单测时，只初始化一次容器，加快了单测执行的速度和效率。

实践及遇到的问题

今天在用Junit进行单元测试的时候遇到了一个问题，测试是pass的，但是总没有出现我要的结果。单步调试的时候会在某一步突然退出，控制台打印“disconnect from the target JVM”。

到网上搜这个错误，没找到我这种情况的解决办法。最后只能用暴力的老办法了—打日志，能打的地方都打上，然后发现一个现象，程序每次所能执行到的代码行是不一样的，但是运行时间是几乎一样的，脑子嗡的一下，突然明白是怎么回事了。我的代码是在子线程中运行的，但是单测的主线程每次运行完就终止了，而不会等待子线程运行完，从而终止了子线程的运行。

又到网上一查，发现Junit是不支持执行多线程测试的。那要怎么办呢？一个最直接的办法就是让单测主线程等待子线程运行完才退出，即sleep一段时间，但也很显然，缺点是主线程不知道要等待多久，所以只能尽可能等待地久一点。这种方法很不优雅，另一种方法就是利用同步结构工具来同步单测主线程和子线程。
